Masalah Web Server

13 Desember, 2024 · 2-3 menit

Kita punya banyak sekali server web. Beberapa untuk produksi, lainnya untuk pra-produksi, dan masing-masing punya keunikan sendiri. Di atas itu semua, yang membuat makin rumit, URL-URL tersebar di berbagai dokumen internal, pesan chat, dan kadang-kadang di bookmark browser.

Pada awalnya, saya mencoba menggunakan bookmark untuk mengatur semuanya, tetapi dengan cepat saya sadar itu tidak cukup. Bookmark bergantung pada lingkungan browser—beberapa server internal kami hanya bisa diakses melalui browser aman tertentu. Jika saya membuka tautan yang salah di browser yang salah, saya bisa mendapat error atau bahkan lebih parah lagi, sesi tidak bisa dimulai sama sekali.

Jadi, saya memutuskan untuk membangun sesuatu yang lebih baik. Saya membuat sebuah portal akses web terpadu, sepenuhnya ditulis dalam HTML sederhana. Bayangkan seperti dashboard, tetapi terinspirasi dari estetika minimalis imageboard seperti 4chan.

Alih-alih mencari di antara bookmark, sekarang saya cukup membuka satu halaman, klik grup aplikasi yang relevan—misalnya, "Layanan Pembayaran – Prod" atau "User API – Preprod"—dan langsung menuju halaman login.

Sejujurnya, ini juga sesuai dengan cara kerja otak saya. Meski enggan mengakuinya, saya bukan seorang multitasker yang hebat. Semakin banyak beban kognitif yang bisa saya hilangkan, semakin baik.
